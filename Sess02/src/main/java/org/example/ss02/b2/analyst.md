| #  | Vị trí                                                                      | Loại vấn đề                                                  | Mô tả chi tiết hậu quả                                                                                                                                                                                                                         |
| -- | --------------------------------------------------------------------------- | ------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1  | `<%! private int requestCounter = 0; %>`                                    | **Race Condition / Shared State**                            | Biến khai báo bằng `Declaration <%! %>` trở thành **biến instance của Servlet JSP**, dùng chung cho mọi request và mọi user. Nhiều người truy cập cùng lúc có thể làm tăng sai giá trị do tranh chấp luồng. Đây là lỗi thiết kế rất nguy hiểm. |
| 2  | Khối `<% requestCounter++; ... %>`                                          | **Logic Java đặt trong View**                                | JSP chỉ nên hiển thị dữ liệu. Việc tăng biến đếm, ép kiểu, lấy dữ liệu từ request bằng Java làm View không còn “ngốc nghếch” nữa. Code khó bảo trì, khó test, vi phạm nguyên tắc **Thin View**.                                                |
| 3  | `List<Student> list = (List<Student>) request.getAttribute("studentList");` | **Scriptlet + ép kiểu thủ công**                             | JSP đang tự lấy dữ liệu từ request và cast kiểu. Điều này làm code dài dòng, dễ lỗi `ClassCastException` hoặc `NullPointerException` nếu attribute thiếu. Đúng chuẩn thì dùng `${studentList}` với JSTL/EL.                                    |
| 4  | Vòng lặp `for (int i = 0; i < list.size(); i++)`                            | **Dùng Scriptlet thay vì JSTL**                              | Việc dùng Java loop trong JSP làm file khó đọc, lẫn logic xử lý với giao diện. Chuẩn JSP view hiện đại là dùng `<c:forEach>` thay thế.                                                                                                         |
| 5  | Khối `if/else` tính `rank` trong JSP                                        | **Business Logic trong View**                                | Logic xếp loại đang nằm ngay trong JSP. Dù chạy được, nhưng đây là code smell rõ ràng. Nếu quy tắc xếp loại thay đổi, phải sửa ngay trong view. Trong bài này yêu cầu refactor bằng JSTL `<c:choose>` để giảm Java code trong JSP.             |
| 6  | `<td><%= sv.getFullName() %></td>`                                          | **XSS**                                                      | In trực tiếp dữ liệu có nguồn từ người dùng mà không escape HTML có thể gây XSS. Ví dụ tên chứa `<script>...</script>`. Cần dùng `<c:out>` để escape an toàn.                                                                                  |
| 7  | `<td><%= sv.getScore()%>;</td>`                                             | **Sai cú pháp hiển thị / ký tự thừa**                        | Dấu `;` nằm ngoài expression nên sẽ bị in ra giao diện thành `92;`, `75;`... Không làm JSP crash nhưng hiển thị sai dữ liệu. Đây là lỗi trình bày rất hay gặp.                                                                                 |
| 8  | `<!-- Tiêu đề trang báo cáo -->`                                            | **Dùng HTML Comment thay vì JSP Comment cho ghi chú nội bộ** | HTML comment vẫn được gửi xuống trình duyệt và người dùng có thể xem source. Nếu là ghi chú nội bộ cho dev thì phải dùng `<%-- --%>` để server bỏ hẳn khỏi response.                                                                           |
| 9  | `<%= title %>`, `<%= requestCounter %>`, `<%= i + 1 %>`...                  | **Dùng Expression cũ thay vì EL**                            | JSP Expression `<%= %>` là kiểu viết cũ, làm code xấu và khó phối hợp với JSTL. Chuẩn hiện nay là dùng `${}` để truy xuất dữ liệu model.                                                                                                       |
| 10 | Directive page import Java model ngay trong JSP                             | **Phụ thuộc Java class trực tiếp trong View**                | `import="java.util.List, com.demo.model.Student"` làm JSP dính chặt với lớp Java. Khi dùng EL + JSTL đúng cách, thường không cần import kiểu này nữa. View sẽ gọn hơn.                                                                         |
